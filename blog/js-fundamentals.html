What is control flow and loops?

Control flow in programming is the order in which things are executed. Comparing this to everyday life you might compare this to a morning routine. You wake up, eat breakfast, shower, brush teeth etc. Each of these steps is taken in an order but you don’t repeat any of the steps because who has two showers in the morning?

Loops in programming is a piece of code or method that is repeated a number of times. The amount of loops is decided by the programmer. We can compare this to eating a meal. You grab the food with your utensil, put it in your mouth, chew, swallow and repeat until the loop until the food is gone. The amount of times you repeat this loop is determined by the amount of food on your plate.

What is the DOM? How do I use it?

The DOM (Document Object Model) is created by the browser when a web page is loaded. It represents the structure of HTML like a tree structure and each branch or leaf in the tree represents an HTML element. The DOM is used to interact with these elements and can be coded via JavaScript. Using the DOM you can:

  • Change and remove HTML elements within the webpage
  • Add or change CSS styling in elements
  • Add event listeners to elements (think button clicks or keystrokes)
  • Create new HTML elements to add to your webpage
  • Form submission in the front end

An simple example of when you might interact with the DOM is when you are wanting to change the CSS styling of your h1 headings. To do this you would use query selector to locate all h1 elements then use a forEach loop to update each header by your chosen styling input.

The difference between accessing Arrays and Objects

Although similar Arrays and Objects are accessed very differently. When you store information in Arrays you are generally storing one type of data, for example cars:

let cars = ['toyota', 'mazda', 'ford', 'honda']

These cars above are stored against a number in the array starting at 0,1,2 and so on. To access the data in this array you would use “cars[0]” which accesses the ‘toyota’ car since that is sitting in position 0 in the array.

Objects are more useful when you want to store multiple data types for one particular objects. I’ll use the car example but expand on that:

let cars = { toyota: { color: "blue", topSpeed: "100km", doors: "4" }, mazda: { color: "red", topSpeed: "150km", doors: "2" } }

The data points of color, top speed, doors are inside of the object car brand (toyota or mazda) and those objects are stored against a larger object "cars". To access the data points you use cars.toyota.color. Objects make it much easier to store various points of data.

What are functions and why are they useful?

JavaScript functions are a block of code that contains an action or method inside of them. For example you may have a function that calculates the average sum of some numbers. You may want that average function to calculate the average age of sports players and in a different section calculate their average scores. Instead of writing code/functions for both player age and scores you can call the one method without needing to calculate averages twice.

Once functions are written they can be used as many times as needed and you can call the function at various points in your code. They help to reduce code and keep applications maintainable.
